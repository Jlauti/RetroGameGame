## Objective

Define the first real Nebula enemy combat contract so enemies create readable pressure, react like opponents instead of props, and can shoot back without collapsing the current loop into chaos.

Required outcomes:
- enemy movement and targeting should feel deliberate enough to create pressure without turning into omniscient aimbot behavior
- enemy return fire should be readable, dodgeable, and compatible with the existing third-person terrain/boundary combat space
- enemy combat roles should be specific enough for runtime implementation and encounter tuning
- the contract must define how enemy aggression interacts with direct fire, ricochet play, terrain-following flight, and current run-failure rules
