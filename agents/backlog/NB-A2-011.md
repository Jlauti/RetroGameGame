## Objective

Implement the approved Nebula combat contract for terrain-follow flight, tangible boundaries, and projectile ricochet in a single runtime pass.

Required outcomes:
- add terrain-follow visual response so the ship reads as skimming terrain contours while preserving current direct control
- make boundaries readable and tangible in runtime
- keep traversal surfaces non-lethal
- make soft pressure boundaries repel, steer, or pressure the player without ending the run
- keep only explicit hard blockers as run-ending crash surfaces
- replace the current "all walls/extrusions are basically the same crash object" behavior with approved boundary-role handling
- make hex extrusions read as intentional lane-shaping geometry instead of incidental crash noise
- keep direct shots as the main baseline attack
- make ricochets deliberate, readable, and rewarding instead of chaotic
- limit ricochet-enabled surfaces to the approved boundary and extrusion roles
- tune extrusion interaction so these objects are useful firing geometry, not just obstacles
- tighten projectile readability at launch, bounce, and impact
- prevent infinite or near-infinite bounce loops, unreadable speed escalation, random-feeling ricochet outcomes, and ricochet rewards that overshadow direct fire
- keep camera, cursor aiming, and movement readability stable
