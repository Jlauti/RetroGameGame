## Objective

Define the visual language packet for Nebula's procedural neon environment so runtime implementation can build readable terrain, cage, and breakable hazards without waiting on modeled environment kits.

Required outcomes:
- establish a clear visual grammar for traversal-safe terrain, structural terrain, ricochet banks, side pockets, breakable hazards, and the side cage
- define how procedural neon environment pieces differ from human-modeled ship actors in silhouette, emissive treatment, and motion language
- define systemic animation/explosion/transformation language for environment hits, breaks, pulses, and boundary pressure
- keep the environment abstract/procedural-first while preserving combat readability from third-person gameplay camera
